# AI Trust Receipt — Candidate v0.1.0

This release introduces the AI Trust Receipt: a candidate governance specification and executable reference implementation for creating durable, user-legible, machine-verifiable records of consequential AI actions.

The package operationalizes five linked controls: a pre-execution authority and evidence gate; a canonical event record; human-readable disclosure derived from that same record; noncompensatory R1–R12 conformance assessment; and an operational remedy reference established before execution.

## Included in this release

- Practitioner workbook and conformance protocol
- Normative candidate technical specification in PDF and Markdown
- Python reference implementation and installable wheel
- Five JSON Schema Draft 2020-12 contracts
- Machine-readable conformance and decision profiles
- Positive and negative conformance vectors
- Human-readable and machine-readable example receipts
- Verification evidence, release manifest, and SHA-256 digest ledger

## Candidate-status boundary

This package proposes an implementation model. It is not an adopted standard, certification program, legal-compliance determination, external audit, or regulatory approval. The included digest demonstrates artifact integrity; it does not authenticate the issuer or establish that the recorded action was fair, lawful, safe, or correct.
